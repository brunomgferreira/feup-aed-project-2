◆ essentialAirports()

void Data::essentialAirports ( )

Identifies and displays the essential airports in the given graph.

This method determines the essential airports in the graph, considering the connectivity of the airports. It performs a depth-first search (DFS) traversal to identify the strongly connected components, and then calculates the essential airports based on the existence of multiple strongly connected components. The results include the number of essential airports.

Time complexity: O(V + E), where V is the number of vertices (airports) and E is the number of edges (flights).

unordered_set< string >  airlineSet,

Finds the best flights between two locations considering various constraints.

This method processes flights between specified origin and destination locations. It uses a BFS algorithm to find the shortest paths, taking into account airline constraints, unwanted airlines, and minimizing the number of different airlines. The result includes the best flight paths based on the specified criteria.

Parameters
originLocationThe origin location for the flights.
destinyLocationThe destination location for the flights.
airlineSetA set of airline codes to consider (constraints) for the flights.
unwantedAirlinesIndicates whether unwanted airlines should be excluded from the flights.
minimizeAirlinesIndicates whether the number of different airlines should be minimized.
Returns
The best flight paths based on the specified criteria.

Time complexity: O(F * E * A), where F is the number of flights, E is the average number of edges (connections) per airport, and A is the average number of airlines per connection.

◆ maximumTrip()

void Data::maximumTrip ( )

Finds and displays the maximum trip in terms of the number of flights between two airports.

This method performs a BFS traversal from each airport to find the maximum trip, defined as the maximum number of flights between two airports. It keeps track of the airports involved in the maximum trip and prints the results, including the number of flights and the sequence of airports in the trip.

Time complexity: O(V * (V + E)), where V is the number of vertices (airports) and E is the number of edges (flights).

◆ minimalAirlines()

pair< list< list< string > >, int > Data::minimalAirlines ( const list< list< string >> &  flights) const
private

Finds the minimal number of airlines required for a set of flights.

This method takes a list of flights, where each flight is represented as a list of airport codes, and determines the minimal number of different airlines required to cover the entire set of flights. It uses a greedy approach, considering each segment of the flights and selecting the airlines that have not been used previously. The result includes the minimal set of flights and the corresponding number of different airlines.

Parameters
flightsA list of flights, where each flight is represented as a list of airport codes.
Returns
A pair containing the minimal set of flights and the number of different airlines required.

Time complexity: O(F * E * A), where F is the number of flights, E is the average number of edges (connections) per airport, and A is the average number of airlines per connection.

const unordered_map< string, pair< list< string >, int >> &  airportTrack

private

Recursively processes the flight paths from origin to the specified destiny based on BFS results.

This method recursively generates flight paths from the specified origin to the given destiny using information obtained from BFS traversal. It retrieves the list of origins for the destiny and constructs flight paths by appending the destiny to each origin's flight path.

Parameters
destinyThe airport code of the destination airport.
airportTrackA map containing information about airports and their respective BFS results.
Returns

Comparator function for sorting airports based on the number of flights in descending order.

This function compares two pairs containing airport codes and the corresponding number of flights. It is used as a custom comparator for sorting airports in descending order based on the number of flights. If the number of flights is the same, the airport codes are compared in lexicographical order.

Parameters
aThe first pair to compare.
bThe second pair to compare.
Returns
True if the number of flights in pair 'a' is greater than that in pair 'b', or if the number of flights is equal, and the airport code in pair 'a' is lexicographically less than that in pair 'b'. False otherwise.

Time complexity: O(1), as it performs a constant number of comparisons.
